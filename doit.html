PRODUCT IDEA

I designed DOIT to turn limited user input into a practical next step.

DOIT combines a conversational support chatbot with automatic task generation, a calendar, notifications, daily check-ins, and a Pomodoro timer with calming sounds.

I wanted the app to create useful structure from a small amount of input while keeping the interface supportive and easy to use.

02

FEATURE MODULES

01 / CONVERSATION

Conversational support chatbot

The chatbot uses Anthropic’s Claude API for conversation, motivation, support, and next-step suggestions.

02 / PLANNING

Smart to-do generation

The app turns a small amount of user input into structured tasks and actionable plans.

03 / TIME

Calendar + notifications

A custom JavaScript calendar and browser notifications keep tasks and commitments inside the same app.

04 / FOCUS

Pomodoro environment

A custom Pomodoro timer includes calming sounds for focused work sessions.

05 / CONTINUITY

Daily check-ins

Daily prompts let users record how they are doing and review task progress.

06 / DATA

Firebase persistence

Firebase stores user and application data.

WHAT I LEARNED

LLM OUTPUT QUALITY

Supportive language does not guarantee a useful task plan.

I learned that prompts need to define structure, tone, task size, and follow-up behavior instead of only asking the model to be helpful.

PRODUCT STRUCTURE

The features need to work as one system.

The chatbot, calendar, tasks, check-ins, and focus tools need consistent navigation and shared data instead of functioning as unrelated features.

SCOPE

This is a productivity prototype, not a clinical tool.

DOIT demonstrates the product concept and interface architecture. I do not present it as a mental-health treatment or clinical application.
